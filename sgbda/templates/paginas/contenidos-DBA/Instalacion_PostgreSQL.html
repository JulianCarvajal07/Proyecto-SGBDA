Guía Instalación PostgreSQL

Esta guía presenta el paso a paso para la instalación de PostgreSQL sobre Oracle Linux 9 / 10, utilizando el repositorio oficial PGDG. A continuación se mostrará el paso a paso a tener presente:

01

Actualizar el Sistema Operativo

Antes de instalar cualquier paquete, se debe actualizar el sistema operativo. Al ser Oracle Linux 9 / 10 un sistema basado en RHEL, el gestor de paquetes a utilizar es dnf y no apt (este último corresponde a distribuciones basadas en Debian).

Actualizar_Sistema.sh 

sudo dnf update -y
02

Agregar el Repositorio Oficial de PostgreSQL (PGDG)

Los repositorios por defecto del sistema operativo no siempre traen la versión de PostgreSQL deseada, por lo cual se agrega el repositorio oficial PGDG (PostgreSQL Global Development Group), que permite elegir e instalar versiones específicas del motor.

Repositorio_PGDG_ORACLE_LINUX_10.sh 

sudo dnf install -y https://download.postgresql.org/pub/repos/yum/reporpms/EL-10-x86_64/pgdg-redhat-repo-latest.noarch.rpm
Repositorio_PGDG.Repositorio_PGDG_ORACLE_LINUX_9.sh 

sudo dnf install -y https://download.postgresql.org/pub/repos/yum/reporpms/EL-9-x86_64/pgdg-redhat-repo-latest.noarch.rpm

Deshabilitar Módulo del Sistema (Opcional)

- En versiones antiguas de RHEL/OL, el sistema trae un módulo de PostgreSQL propio que entra en conflicto con el repositorio PGDG y debe deshabilitarse.

- En versiones recientes del sistema operativo (como OL10) no existe dicho módulo, por lo tanto no hay conflicto activo y este paso puede omitirse.

Deshabilitar_Modulo.sh 

sudo dnf -qy module disable postgresql
03

Instalar una Versión Específica de PostgreSQL

Se instala la versión del motor requerida, indicando explícitamente el número de versión en el nombre del paquete.

Buenas Prácticas

- Siempre fijar la versión del paquete a instalar, evitando el uso de latest en ambientes de producción.

- Versiones recomendadas actualmente: 15 y 16.

Instalar_PostgreSQL.sh 

-- Ejemplo instalando la version 16
sudo dnf install -y postgresql16-server postgresql16
04

Crear Estructura en Disco e Inicializar el Clúster

Se verifica que el binario de instalación exista, se crea la estructura de directorios en el disco dedicado para las bases de datos, se ajustan los permisos correspondientes al usuario postgres y finalmente se inicializa el clúster con initdb.

Verificar Binarios

Confirmar que existan los archivos de instalación (initdb, entre otros) en la ruta /usr/pgsql-16/bin antes de continuar.

Permisos y Propietario

El directorio destinado a los datos debe pertenecer al usuario postgres y tener permisos restringidos (700), ya que este usuario es quien administra el motor.

Crear_Estructura_e_Inicializar.sh 

-- Verificar si hay archivos en esta ruta (initdb, etc)
ls /usr/pgsql-16/bin

-- Creación de estructura en el disco dedicado para BD
sudo mkdir -p /var/lib/postgresql/16/main
sudo chown -R postgres:postgres /var/lib/postgresql
sudo chmod 700 /var/lib/postgresql/16/main

-- Inicializar el clúster correctamente
sudo -u postgres /usr/pgsql-16/bin/initdb -D /var/lib/postgresql/16/main
05

Configurar e Iniciar el Servicio

Si al iniciar el servicio se presenta algún error, se debe verificar el archivo de configuración del servicio systemd, validando que la ruta del clúster configurada coincida con la creada en el paso anterior. Luego se procede a iniciar el servicio y dejarlo habilitado para que inicie junto con el sistema operativo.

Configurar_Servicio.sh 

-- Si existe un error, verificar la ruta configurada en el servicio
sudo nano /usr/lib/systemd/system/postgresql-16.service

-- hacer que systemd vuelva a leer los archivos de configuración de los servicios.
sudo systemctl daemon-reload

-- Iniciar el servicio de PostgreSQL
sudo systemctl start postgresql-16

-- Verificar que el servicio este corriendo
sudo systemctl status postgresql-16

-- Para que inicie automaticamente con el sistema
sudo systemctl enable postgresql-16
06

Configurar el Usuario postgres y la Autenticación

Por defecto se crea el usuario postgres, el cual cumple un rol equivalente al usuario root en MariaDB. Este usuario no tiene contraseña por defecto, ya que utiliza autenticación del sistema (Linux) a través del método peer.

Asignar Contraseña

Ingresar como el usuario postgres del sistema operativo, abrir la consola psql y asignar una contraseña al usuario postgres de la base de datos.

Habilitar Autenticación por Contraseña

Una vez asignada la contraseña, se debe editar el archivo pg_hba.conf para cambiar el método de autenticación local del usuario postgres, pasando de peer a md5, habilitando así el ingreso por contraseña (socket).

Configurar_Usuario_Postgres.sh 

-- Ingresar como el usuario postgres del sistema operativo
sudo -i -u postgres
psql

-- Nota: el usuario postgres no tiene contraseña por defecto
-- si el usuario es postgres, usa autenticacion del sistema (peer)

-- Asignar contraseña al usuario postgres
ALTER USER postgres WITH PASSWORD 'N2tgr45psa';
pg_hba.conf 

-- Ruta del archivo a editar (segun la version instalada)
nano /var/lib/postgresql/16/main/pg_hba.conf

-- Cambiar la siguiente linea o comentar
# local   all             all                                     trust
# local   all             postgres                                peer

-- Por la siguiente
local   all             postgres                                md5

-- Se Reinicia el Servicio de Postgresql-16
sudo systecmtl restart postgresql-16
07

Conectarse desde pgAdmin

Como último paso, se valida la conexión al motor recién instalado utilizando pgAdmin, ingresando los datos configurados en los pasos anteriores.

postgresql.conf 

-- Ruta del archivo a editar (segun la version instalada)
nano /var/lib/postgresql/16/main/postgresql.conf

-- Se cambia la linea:
listen_addresses = 'localhost' 

-- por:
listen_addresses = '*'

-- luego ir a la ruta:
nano /var/lib/postgresql/16/main/pg_hba.conf

-- anexar la siguiente linea para conexion remota:
host    all             postgres        0.0.0.0/0               md5

-- reiniciar el servicio:
sudo systemctl restart postgresql-16
Configuracion_firewall.ssh 

-- Se habilita puerto 5432
sudo firewall-cmd --add-port=5432/tcp --permanent
sudo firewall-cmd --reload

-- Se verifica la configuracion del firewall
sudo firewall-cmd --list-all

-- se verifica la comunicacion con el host y puerto
ping (IP_MAQUINA)
telnet (IP_MAQUINA) 5432

habilitar el puerto en el firewall es necesario si se instala postgresql en una distribucion basada en Red Hat

Datos de Conexión

- Host: (IP_MAQUINA) / localhost

- Usuario: postgres (o el usuario que se haya creado)

- Password: la contraseña configurada en el Paso 06

- Puerto: 5432
