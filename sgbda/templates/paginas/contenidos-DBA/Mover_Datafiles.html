Como Mover Datafiles de un Disco a Otro

Esta guía presenta el paso a paso para mover los archivos físicos de una base de datos en SQL Server: los archivos .mdf y .ndf hacia el Disco D y el archivo .ldf hacia el Disco L, garantizando que no se pierda información y que la ventana de indisponibilidad sea mínima. A continuación se mostrará el paso a paso a tener presente:

01

Identificar Archivos Lógicos y Rutas Actuales

Antes de mover cualquier archivo, se debe identificar el nombre lógico de cada datafile y la ruta física en la que se encuentra actualmente. Esta información es indispensable, ya que el nombre lógico es el que se usará para indicarle a SQL Server la nueva ubicación del archivo.

Consultar desde T-SQL

Ejecutar el procedimiento sp_helpfile sobre la base de datos, o consultar la vista sys.master_files para obtener el nombre lógico, la ruta física y el tipo de archivo (ROWS o LOG) de cada datafile.

Consultar desde SSMS

Alternativamente, ingresar a Object Explorer > clic derecho sobre la base de datos > Properties > pestaña Files, allí se listan los nombres lógicos y rutas de cada archivo.

Identificar_Archivos.sql 

                -- Ver nombre lógico, tipo y ruta física de cada archivo de la base de datos
                USE [NOMBRE_BASE_DE_DATOS]
                GO

                SP_HELPFILE

                -- Alternativa usando catálogo del sistema
                SELECT
                    DB_NAME(database_id) AS BaseDatos,
                    name                  AS NombreLogico,
                    type_desc             AS TipoArchivo,   -- ROWS = MDF/NDF, LOG = LDF
                    physical_name         AS RutaActual
                FROM sys.master_files
                WHERE database_id = DB_ID(N'NOMBRE_BASE_DE_DATOS');
02

Redireccionar la Ruta Lógica del Archivo

Con el comando ALTER DATABASE ... MODIFY FILE se le indica a SQL Server la nueva ruta física que tendrá cada archivo. Este comando puede ejecutarse con la base de datos en línea, ya que solo actualiza el metadato de la ruta; el cambio se hará efectivo únicamente cuando la base de datos vuelva a iniciar (quede OFFLINE y luego ONLINE). Esto permite adelantar trabajo sin generar indisponibilidad todavía.

Reglas

- Los archivos .mdf y .ndf (tipo ROWS) se redireccionan al Disco D.

- El archivo .ldf (tipo LOG) se redirecciona al Disco L.

- El NAME usado debe coincidir exactamente con el nombre lógico obtenido en el paso anterior.

- Verificar previamente que exista espacio suficiente en el disco destino y que la carpeta de destino ya esté creada.

Redireccionar_Rutas.sql 

                USE [master]
                GO

                -- Redireccionar archivo(s) de datos (.mdf / .ndf) al Disco D
                ALTER DATABASE [NOMBRE_BASE_DE_DATOS]
                MODIFY FILE (NAME = N'NOMBRE_LOGICO_MDF', FILENAME = N'D:\SQLData\NOMBRE_ARCHIVO.mdf');

                ALTER DATABASE [NOMBRE_BASE_DE_DATOS]
                MODIFY FILE (NAME = N'NOMBRE_LOGICO_NDF', FILENAME = N'D:\SQLData\NOMBRE_ARCHIVO.ndf');

                -- Redireccionar archivo de log (.ldf) al Disco L
                ALTER DATABASE [NOMBRE_BASE_DE_DATOS]
                MODIFY FILE (NAME = N'NOMBRE_LOGICO_LDF', FILENAME = N'L:\SQLLog\NOMBRE_ARCHIVO.ldf');
03

Poner la Base de Datos Fuera de Línea

A partir de este punto inicia la ventana de indisponibilidad. Se debe colocar la base de datos en estado OFFLINE para poder mover los archivos físicos sin que se encuentren en uso por el motor de SQL Server. Se recomienda validar previamente que no existan transacciones activas ni sesiones abiertas contra la base de datos.

Poner_Offline.sql 

                USE [master]
                GO

                -- Cerrar conexiones activas y dejar la base de datos fuera de línea
                ALTER DATABASE [NOMBRE_BASE_DE_DATOS]
                SET OFFLINE WITH ROLLBACK IMMEDIATE;
04

Mover Físicamente los Archivos

Con la base de datos OFFLINE, se procede a mover (no copiar) los archivos desde el Disco C hacia sus nuevos destinos. Es importante cortar y pegar o usar una utilidad que mueva el archivo, evitando dejar duplicados en el disco de origen que consuman espacio innecesariamente.

Tambien se debe asignar PERMISO TOTAL al usuario sobre el disco para realizar dicha maniobra:

Desde el Explorador de Archivos

Cortar los archivos .mdf y .ndf desde la ruta actual en el Disco C y pegarlos en la ruta configurada en el Disco D (por ejemplo D:\SQLData\). Cortar el archivo .ldf y pegarlo en la ruta configurada en el Disco L (por ejemplo L:\SQLLog\).

Desde Línea de Comandos (opcional)

Se puede usar robocopy con el parámetro /MOV para mover el archivo conservando permisos, o el comando move de CMD para una migración rápida dentro de la misma máquina.

Mover_Archivos.cmd 

                :: Mover archivos de datos al Disco D
                move "C:\SQLData\NOMBRE_ARCHIVO.mdf" "D:\SQLData\NOMBRE_ARCHIVO.mdf"
                move "C:\SQLData\NOMBRE_ARCHIVO.ndf" "D:\SQLData\NOMBRE_ARCHIVO.ndf"

                :: Mover archivo de log al Disco L
                move "C:\SQLLog\NOMBRE_ARCHIVO.ldf" "L:\SQLLog\NOMBRE_ARCHIVO.ldf"
05

Poner en Línea y Validar la Información

Una vez movidos los archivos a las nuevas rutas, se coloca la base de datos nuevamente ONLINE. En este momento SQL Server buscará los archivos en las rutas configuradas en el Paso 02; si la ruta y el nombre coinciden, la base de datos abrirá sin pérdida de información, con lo cual finaliza la ventana de indisponibilidad.

Poner_Online_Y_Validar.sql 

                USE [master]
                GO

                -- Poner la base de datos nuevamente en línea
                ALTER DATABASE [NOMBRE_BASE_DE_DATOS]
                SET ONLINE;
                GO

                -- Validar que las rutas físicas ya correspondan al Disco D y Disco L
                SELECT
                    DB_NAME(database_id) AS BaseDatos,
                    name                  AS NombreLogico,
                    type_desc             AS TipoArchivo,
                    physical_name         AS RutaActual,
                    state_desc            AS Estado
                FROM sys.master_files
                WHERE database_id = DB_ID(N'NOMBRE_BASE_DE_DATOS');

                -- Validar el estado general de la base de datos
                SELECT name, state_desc
                FROM sys.databases
                WHERE name = N'NOMBRE_BASE_DE_DATOS';

Puntos de Validación

- El campo state_desc de sys.databases debe mostrar ONLINE.

- Las rutas en physical_name deben apuntar ya al Disco D (MDF/NDF) y al Disco L (LDF).

- Revisar el Log de Errores de SQL Server (SSMS > Management > SQL Server Logs) para confirmar que no se generaron errores al montar los archivos.

- Realizar una consulta rápida (por ejemplo SELECT TOP 10 * FROM alguna tabla) para confirmar que la información se conserva intacta.
